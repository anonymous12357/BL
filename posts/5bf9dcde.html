VPS内快速检测Google服务地区与油管视频缓冲节点地区

vultr、linode之类部分ip会被Google当成阿联酋，全是清真文。开了机就测试，省去很多麻烦

Google服务地区
curl https://www.有图比.com/red | sed ‘s/,/\n/g’ | grep countryCode
偶尔会无结果，多试几次

油管视频缓冲节点地区
curl https://redirector.googlevideo.com/report_mapping | grep router
如果无返回结果，则说明是使用运营商与Google合作的Google Global Cache节点，大部分速度不好
用下面这个查看
curl https://redirector.googlevideo.com/report_mapping | grep “=>”
已知这些
AS2514 NTTPC
AS2516 KDDI
AS2518 Biglobe
AS2527 Sony
AS10013 FreeBit
AS17511 Kopticom

例：
vultr日本东京43.224.35.*
“countryCode”:”US” 服务地区美国
router: “pr06.lax16” 视频缓冲美国洛杉矶
跨国缓冲速度慢，垃圾

vultr日本东京45.76.53.*
“countryCode”:”US” 服务地区美国
router: “pr04.nrt19” 视频缓冲日本东京
不影响缓冲速度，一般

vultr日本东京45.76.194.*
“countryCode”:”PH” 服务地区菲律宾
router: “pr04.nrt19” 视频缓冲日本东京
不影响缓冲速度，但地区比较坑，一般偏下

vultr日本东京108.61.201.*
“countryCode”:”JP” 服务地区日本
router: “pr04.nrt19” 视频缓冲日本东京
良好

原理 -> VPS内快速检测服务地区

分析:

ipv6解析到了日本的缓存服务器，ipv4解析到了南非的缓存服务器。

而默认是ipv6优先，通过服务器端curl ip.sb查看的确是ipv6。

可以通过修改/etc/gai.conf修改ipv4优先，但某些情况下实际使用可能无效。

不如干脆一点，把ipv6给关了。编辑/etc/sysctl.conf，添加以下内容。

net.ipv6.conf.all.disable_ipv6=1
net.ipv6.conf.default.disable_ipv6=1
net.ipv6.conf.lo.disable_ipv6=1
net.ipv6.conf.eth0.disable_ipv6=1

然后运行sysctl -p，再curl ip.sb变成了ipv4。

浏览器连上之后打开 我的IP 检查是不是ipv4(塞舍尔)

补充一个可能有用的提速帖子 -> 一招提升速度

今天刚学会了一招，分享给MJJ们，打开命令行工具，输入netsh interface tcp set heuristics disabled，回车，v2的速度直接起飞，win7亲测有用。

-——————————————————

事情起因是，前几天白漂的V开头的VPS终于不pending了，排除万难后装上了v2，结果打开有图比9000的速度，和我那两个甲骨文日本的表现一样，虽然够用了，但还是有点失望，好奇用手机试了试，意外发现手机比电脑要快，手机speedtest测速有50m，电脑上测顶多20m，带着手机比电脑快的问题百度无果，换谷歌有了果。

从这个帖子 https://github.com/shadowsocks/shadowsocks-windows/issues/1875 跑到了 https://www.duckware.com/blog/how-windows-is-killing-internet-download-speeds/index.html 这个帖子，按老外说的试了试，真的直接起飞了，甲骨文50000站起来了，V开头的罗马尼亚小鸡也也能慢慢爬到3-4m的样子，老外分析东西挺有一手，具体是什么原因你们可以去原帖看。

看来以后不能老是怨小鸡不行，还是得从自身出发，好好提升自己的姿势水平啊。。
